Privacy Policy

Effective date: 3 July 2026

Spike Publisher ("the tool", "we") is an in-house publishing tool built and operated by a single individual — the creator of the Spike channel (the "operator"). It is used solely to upload the operator's own finished videos to the operator's own accounts on YouTube, TikTok and, prospectively, Instagram. It is not a service offered to the public and has no accounts or users other than the operator.

This policy explains, honestly, the small amount of data the tool touches and what it deliberately does not do.

1. Who this applies to

This tool is a single-operator tool. It authenticates only against accounts owned by the operator. There are no end users, no visitor accounts, and no third parties whose data is processed. If you are visiting this website, we collect nothing about you (see section 5).

2. Data the tool accesses

  • OAuth access tokens for the operator's own YouTube, TikTok and Instagram accounts. These are obtained through each platform's official authorization flow and are used only to upload and schedule the operator's videos. Tokens are stored locally on the operator's own machines and are never transmitted to any third party.
  • Video files and their metadata — the finished videos created by the operator, together with titles, descriptions, captions, thumbnails, publish times and AI-content disclosure flags. This content is authored by the operator; it does not contain third-party personal data.
  • Platform API responses returned during upload — for example the resulting video ID, upload status and scheduling confirmation — which are used to verify that a publish succeeded and are stored locally in the operator's own logs.

3. How that data is used

The accessed data is used for one purpose only: to stage, schedule, publish and verify the operator's own videos on the operator's own accounts. It is not used for advertising, profiling, resale, or any secondary purpose.

4. What the tool does NOT do

  • It does not access, collect, or process data from any third-party users, viewers, or audience members.
  • It does not read analytics, followers, comments, or private data belonging to anyone other than the operator, and never on behalf of a third party.
  • It does not sell, rent, or share any data with third parties.
  • It does not use the platform data for advertising or building user profiles.
  • This website does not set cookies, run analytics, embed third-party trackers, or make external network requests.

5. This website

This website (levoely.com) is a static informational site. It uses no cookies, no analytics, and no third-party scripts, fonts, or embeds. Visiting it does not create any record about you beyond the standard, transient server request logs kept by the static hosting provider, which we do not use or process.

6. Data storage and retention

OAuth tokens, video files, metadata and API-response logs are stored locally on the operator's own machines for as long as they are needed for publishing and record-keeping. They are not uploaded to any external database or shared service operated by us. Tokens can be revoked at any time (see section 7), after which they are removed from local storage.

7. Revoking access

Because the tool uses standard OAuth, the operator can revoke its access to any connected account at any time, directly in the platform's own security settings:

  • TikTok — Profile → Settings and privacy → Security & permissions → Manage app permissions, then remove the app.
  • Google / YouTube — Google Account → Security → "Your connections to third-party apps & services", select the app, then Remove access.
  • Meta / Instagram — Accounts Center / Settings → Apps and websites → Active, select the app, then Remove.

Revoking access immediately invalidates the stored tokens; the operator then deletes the local copies.

8. Children

This is not a consumer service and is not directed to children. No data about any individual, including children, is collected.

9. Changes to this policy

If the tool's data handling changes, this policy will be updated and the effective date above revised. Because there are no users to notify, the current version on this page is always authoritative.
